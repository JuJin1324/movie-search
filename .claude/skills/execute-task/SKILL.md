# Execute Task

## 목적

코딩이면 *Task 하나(PR 단위)*를, 비코딩(문서) Story면 *Story 자체*를 실행하고 결과를 산출물로 남긴다.

## 입력

다음 중 하나:
- **코딩**: `epics/<epic-slug>/3-plan/storyN/tasks.md`의 한 항목 (Task)
- **비코딩**: plan-stories 산출물에서 선택된 *비코딩 Story* 하나 (plan-tasks를 건너뛴 경우)

## 출력

*초안*. 실행 결과·근거·다음 행동을 기록한다.

- **코딩**: `epics/<epic-slug>/3-plan/storyN/taskN-executed.md` — Task 하나당 한 파일
- **비코딩**: `epics/<epic-slug>/3-plan/storyN/storyN-executed.md` — Story 하나당 한 파일

각 산출물은 다음 패턴을 따른다:

- **무엇을 했는가** — Task/Story의 의도와 실제 수행한 작업
- **근거·결정 사항** — 작업 중 내린 결정과 그 이유
- **결과** — 만들어진 산출물 (코드 PR, 문서, 구조 변경 등)
- **(있다면) 남은 일 / 다음 행동** — 후속 단위로 보내야 할 항목, 사용자 검토 포인트

산출물 자체가 완성본이 아니다 — 사용자가 AI 에이전트와 대화하며 채워간다. 스킬은 *진입점*과 *초안*을 제공할 뿐.

## 작동 방식

핑퐁. Task 또는 Story 하나를 끝까지 실행한 뒤 결과를 정리한다.

1. **단위 확인** — 입력이 Task인지 Story인지, 의도·완료 단위·검토 이야기를 사용자와 다시 짚는다.
2. **실행** — 코딩이면 코드 작성·테스트, 비코딩이면 문서/구조 작업. AI가 한 덩어리로 수행하지 않고 *사용자가 따라갈 수 있는 단위*로 분해해 핑퐁.
3. **결과 기록** — 실행이 끝나면 해당 산출물 파일에 *무엇을 했는지·왜 그렇게 했는지·결과·다음 행동*을 정리.
4. **검토** — 사용자가 PR 또는 산출물을 검토. 다음 단위로 갈지, Task/Story 자체가 흔들리면 상위(plan-tasks / plan-stories)로 돌아갈지 결정.

매 단계마다 *작성 → 이유 설명 → 사용자 확인* 순서.

## 경계

- **새 계획은 여기서 안 한다.** 단위 자체가 흔들리면 상위(plan-tasks / plan-stories)로 돌아간다.
- **새 단위 분해는 여기서 안 한다.** 실행 중 더 큰 작업이 발견되면 상위로 보낸다.
- **문제 정의·해결 방안 탐색은 여기서 안 한다.** 입력 단위는 이미 그 위에서 분해된 것이므로 *실행*에만 집중한다.
